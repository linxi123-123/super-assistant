# 项目总控文件

本文件是后续所有任务的最高优先级项目总控文件。

## 1. 项目终局目标

本项目不是聊天助手，不是普通记忆库，不是日报工具，不是待办工具。

本项目是“个人超级军师系统”。

核心定义：

它是一个持续运行的个人局势智能体，通过多源感知理解用户真实处境，通过长期记忆建立动态个人模型，通过外部情报雷达识别机会和风险，通过推理系统形成军师判断，并在关键时刻主动触达用户，最终逐步具备跨工具执行能力。

核心主循环：

```text
感知
-> 原始事件
-> 候选记忆
-> 长期记忆
-> 动态个人模型
-> 外部情报
-> 信号识别
-> 主动触达
-> 用户反馈
-> 结果追踪
-> 模型修正
-> 必要时执行
```

任何功能如果不服务这个主循环，就不要做。

## 2. 产品不可妥协原则

- 不做普通聊天助手。
- 不以聊天框作为产品中心。
- 首页应围绕“个人作战室 / 当前局势”设计。
- 所有判断必须有证据链。
- 所有模型判断必须有置信度、反例和验证方式。
- 系统必须能反迎合，而不是只顺着用户说。
- 主动触达必须有行动价值，不能只是提醒噪音。
- 外部情报必须和用户目标、风险、机会相关，不能做普通新闻摘要。
- 执行动作必须绑定目标、理由、完成标准和结果追踪。
- 判断质量未通过前，不允许固化数据库。
- MVP 可以小，但不能违背“感知 -> 记忆 -> 推理 -> 触达 -> 反馈”的主循环。

## 4. 当前禁止事项

在最终审计通过前，禁止：

- 进入 SQLite。
- 创建 SQLite 本地应用。
- 写数据库迁移代码。
- 引入 npm。
- 引入 React / Vue / Next.js / Electron 等框架。
- 接入浏览器、邮件、语音、健康数据、位置等新数据源。
- 做 UI 美化。
- 做执行代理。
- 做自动发邮件、自动控制电脑等动作。
- 把当前规则直接固化进数据库。
- 为了提高测试分数只硬编码测试原句。

## 5. 进入 SQLite Readiness Review 的门槛

必须同时满足：

- 固定 10 条测试平均分 >= 24/30。
- 变体 5 条测试平均分 >= 24/30。
- hidden 5 条测试平均分 >= 22/30。
- 信号识别平均分 >= 4/5。
- 反迎合能力平均分 >= 4/5。
- 没有高价值信号被降级成 general risk 后不触达。
- 没有明显伪反迎合。
- 没有明显 scope creep 漏判。
- 没有明显测试集过拟合。

如果未全部满足，不允许进入 SQLite readiness review。

## 6. 如果最终审计通过

仍然不要直接开发 SQLite。

只能输出：

- `docs/sqlite_readiness_review.md`
- `tasks/sqlite_migration_plan.md`

然后停止，等待用户确认。

## 7. 如果最终审计未通过

继续留在 V1 最终测试审计阶段或退回 V1 语义泛化补强阶段。

必须输出失败原因：

- 是规则覆盖问题？
- 是 currentPhaseContext 阶段判断问题？
- 是信号评分问题？
- 是触达模板问题？
- 是反迎合不足？
- 是测试集过拟合？
- 是否需要重构 signal pipeline？

不得建议进入 SQLite。

## 防过拟合原则

1. 禁止为任何单条测试句创建专属规则。
2. 每个 semantic cluster 必须描述一类稳定语义，而不是覆盖某个句子。
3. 如果某条规则只能解释 1 条测试输入，视为过拟合，不允许计入有效补强。
4. 每个 semantic cluster 必须包含：
   - 本质定义
   - 正向语义线索
   - 负向样例
   - 非测试验证样例
   - 当前阶段下的触发条件
   - 不应触发的边界
5. 每轮复测必须包含固定测试集和 hidden semantic variants，防止只针对题库优化。

## 反迎合能力强制扣分

1. 如果反方判断是通用套话，与当前事件无直接关系，最高 2/5。
2. 如果反方判断没有指出用户当前可能的具体误判，最高 3/5。
3. 如果反方判断没有结合 currentPhaseContext，最高 3/5。
4. 如果反方判断没有说明不听建议的代价，最高 3/5。
5. 如果只是“先肯定用户，再泛泛提醒风险”，视为伪反迎合，最高 2/5。
6. 如果反方判断能指出用户想做的事为什么不该现在做，并给出更优动作，才允许 4/5 以上。

## 军师提醒强制扣分

1. 如果提醒只是复述输入，最高 2/5。
2. 如果没有当前阶段判断，最高 3/5。
3. 如果没有下一步动作，最高 3/5。
4. 如果没有说明不行动的后果，最高 3/5。
5. 如果没有和长期目标或当前阶段关联，最高 3/5。

## 信号识别强制扣分

1. 如果高价值信号被降级为 general risk，最高 3/5。
2. 如果 high-value signal 没有生成 touchpoint，最高 3/5。
3. 如果 scope creep 没有经过 currentPhaseContext 比对，最高 3/5。
4. 如果 platform competition risk 被当作普通风险，最高 3/5。
5. 如果 judgment quality risk 被当作普通担忧，最高 3/5。
