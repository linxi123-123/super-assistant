# V1 手动判断质量测试

## 测试目标

验证静态原型是否真正跑通并产生有质量的军师主循环，而不是只根据关键词生成机械内容。

测试链路：

```text
真实事件输入
-> 事件卡片
-> 候选记忆
-> 个人模型假设
-> 信号识别
-> 军师提醒
-> 用户反馈
-> 结果追踪
-> 模型修正
```

## 测试输入

### 事件 1

今天 Codex 已经完成了超级军师系统的 V1 架构验收和补强，补了用户反馈、结果追踪、模型修正历史，并验证了静态页面和 SQLite schema。

建议事件类型：行为事件

### 事件 2

我担心现在虽然有了事件、记忆、模型、信号、触达、反馈，但它可能只是根据关键词生成内容，并没有真正理解我的处境。

建议事件类型：风险事件

### 事件 3

我今天又想继续讨论更多终局功能，比如语音、浏览器、邮件、位置和健康数据接入，但还没有测试现在 V1 的 5 条真实事件。

建议事件类型：偏离事件

### 事件 4

我看到越来越多 AI 产品都在做 memory、agent、personal assistant，我担心这个方向很快会被大厂吃掉。

建议事件类型：风险事件

### 事件 5

我决定先不继续扩展功能，也不马上上 SQLite，而是先完成 5 条真实事件测试，并根据结果修改判断规则。

建议事件类型：承诺事件

## 评分维度

每项 0-5 分：

- 事件理解
- 记忆提取
- 模型假设
- 信号识别
- 军师提醒
- 反迎合能力

## 进入下一阶段规则

- 总分低于 24/30：不建议进入 SQLite。
- 信号识别低于 4/5：先补规则。
- 反迎合能力低于 4/5：先补规则。
