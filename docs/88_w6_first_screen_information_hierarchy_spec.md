# W6 第一屏信息层级修正方案

生成时间：2026-06-11

阶段：W6：第一屏当前语境与阶段时效规格修正

## 1. 第一屏新顺序

第一屏建议顺序：

1. 当前阶段标签
2. 判断生成时间 / 判断有效期
3. 局势一句话
4. 军师直接判断
5. 今日唯一动作
6. 不要做清单
7. 当前最大矛盾
8. 下一阶段门槛
9. 历史审计提示

这个顺序的核心原则是：先说明当前语境，再给判断和动作，最后再解释边界与历史审计内容。

## 2. 当前内容与历史内容的分区

第一屏和下方模块必须明确分区，避免把当前动作和历史审计混在一起。

### 当前判断区

当前判断区包含：

- 当前阶段
- 判断时效
- 局势一句话
- 军师直接判断
- 今日唯一动作
- 不要做清单
- 下一阶段门槛

当前判断区只回答“现在该如何理解局势、现在唯一该做什么、现在不该做什么”。

### 历史审计区

历史审计区包含：

- High-value Signals
- Personal Model Hypotheses
- Recent History
- Source Metadata
- Snapshot Audit

历史审计区必须有提示：

```text
以下内容来自历史/测试快照，用于审计，不代表当前唯一行动指令。
```

历史审计区的职责是解释判断来源、保留证据链、支持复盘，而不是生成当前待办。

## 3. 不允许的设计

第一屏不允许：

- 把历史 signals 放到当前动作前面。
- 把多个动作并列展示成待办列表。
- 把长期记忆治理当成当前已实现能力。
- 把长期架构图内容直接展示成当前功能。
- 把 W5B/W5D 的历史动作当成 W6 当前动作。

第一屏也不应把“历史审计材料”包装成新的军师判断。只要内容来自历史快照，就必须被标注为审计或参考。
