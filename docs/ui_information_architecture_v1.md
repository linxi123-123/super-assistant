# UI 信息架构 v1

## 总体原则

首页是个人作战室，不是聊天页。

对话只是入口之一，系统主要通过目标、事件、记忆、模型、信号和触达形成军师主循环。

## 导航结构

```text
作战室
事件
目标与项目
记忆
个人模型
信号
决策
情报
执行
设置
```

## 1. 作战室 Dashboard

首屏模块：

- 当前阶段判断
- 当前核心目标
- 正在推进的项目
- 当前主要风险
- 当前主要机会
- 今日最重要动作
- 近期承诺
- 偏离信号
- 需要确认的记忆
- 军师主动建议

关键设计：

- 不放大聊天框。
- 每个判断都能点开证据链。
- 每个提醒都有“准确 / 不准确 / 稍后处理 / 转成行动”。

## 2. 事件页

功能：

- 新增每日事件。
- 查看事件列表。
- 按事件类型筛选。
- 关联目标和项目。
- 设置重要性、置信度、敏感度。

事件输入字段：

```text
发生了什么？
这属于什么类型？
和哪个目标/项目有关？
重要性是多少？
置信度是多少？
是否敏感？
原始证据是什么？
```

## 3. 目标与项目页

目标区：

- 长期目标
- 中期目标
- 短期目标
- 成功标准
- 失败标准

项目区：

- 项目阶段
- 当前进展
- 关键风险
- 下一步动作
- 关联事件
- 关联记忆
- 关联决策

## 4. 记忆中心

分区：

- 事实记忆
- 目标记忆
- 项目记忆
- 偏好记忆
- 判断记忆
- 承诺记忆
- 担忧记忆
- 行为模式记忆
- 资源记忆
- 结果记忆

每条记忆展示：

- 内容
- 类型
- 来源事件
- 证据
- 置信度
- 重要性
- 敏感度
- 是否确认
- 是否过期
- 是否允许用于建议

操作：

- 确认
- 修改
- 删除
- 禁止用于建议
- 标记过期

## 5. 个人模型页

展示系统对用户的阶段性理解。

模块：

- 当前阶段
- 核心目标
- 主要资源
- 主要约束
- 主要风险
- 最近注意力流向
- 最近真实产出
- 行为模式
- 决策偏差
- 系统假设
- 待验证项

每条模型假设必须显示：

- 假设内容
- 证据
- 反例
- 置信度
- 验证计划
- 用户反馈

## 6. 信号中心

信号类型：

- 偏离信号
- 机会信号
- 风险信号
- 承诺信号
- 决策信号
- 状态信号
- 模式信号

每条信号展示：

- 描述
- 证据
- 相关目标
- 相关项目
- 重要性
- 紧急性
- 目标相关度
- 可行动性
- 置信度
- 打扰成本
- 是否触达

## 7. 决策页

功能：

- 新增重大问题。
- 按 17 步链路推演。
- 保存决策日志。
- 设置复盘时间。

决策结构：

```text
问题定义
真实目标
当前筹码
关键约束
可选路径
收益
代价
成功概率
失败代价
机会成本
反方观点
推荐方案
24 小时动作
7 天动作
30 天动作
验证指标
复盘时间
```

## 8. 情报页

V1 支持手动或半自动录入情报。

每条情报必须回答：

- 这件事是什么？
- 为什么重要？
- 和用户有什么关系？
- 验证或推翻了哪个判断？
- 带来什么机会？
- 带来什么风险？
- 是否需要行动？
- 第一步是什么？

## 9. 执行页

V1 只做到 E0-E1。

功能：

- 查看建议动作。
- 生成执行草稿。
- 查看执行状态。
- 记录结果和复盘。

## 10. 设置页

功能：

- 数据权限开关。
- 执行权限开关。
- 记忆审计。
- 一键删除。
- 模型调用设置。
- 情报源设置。
