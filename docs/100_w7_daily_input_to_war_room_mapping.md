# W7 日常输入到作战室判断映射

生成时间：2026-06-11

## 1. 映射目标

本规格定义每日作战输入包如何映射到作战室第一屏。

目标不是生成更多模块，而是稳定回答五个关键问题：

1. 当前局势一句话。
2. 当前最大矛盾。
3. 军师直接判断。
4. 今日唯一动作。
5. 不要做清单。

## 2. 输入到第一屏字段映射

### 2.1 当前局势一句话

主要来源：

- `key_event`
- `current_question`
- `project_state_change`

生成规则：

- 用一句话概括当前最重要的状态。
- 必须包含用户当前真实处境。
- 不得引用历史测试模块作为当前局势。

### 2.2 当前最大矛盾

主要来源：

- `current_question`
- `project_state_change`
- `constraints_or_forbidden`

生成规则：

- 找出“想推进什么”和“现在不能做什么”之间的冲突。
- 如果矛盾不清楚，应显示“当前最大矛盾待澄清”。

### 2.3 军师直接判断

主要来源：

- `key_event`
- `current_question`
- `evidence_note`
- `use_for_today_judgment`

生成规则：

- 只基于允许参与今日判断的输入。
- 必须有反方判断。
- 必须声明置信度或限制。
- 不得把未确认的长期记忆当成事实。

### 2.4 今日唯一动作

主要来源：

- `committed_action`
- `current_question`
- `constraints_or_forbidden`

生成规则：

- 只输出一个动作。
- 多个动作冲突时，输出“先澄清优先级”。
- 今日动作必须可完成、可验收。

### 2.5 不要做清单

主要来源：

- `constraints_or_forbidden`
- 阶段路线图
- 当前风险判断

生成规则：

- 必须优先显示当前阶段硬边界。
- 不得用历史禁止事项覆盖当前禁止事项。
- 不得允许跳过人工验收或阶段门槛。

## 3. 输入到历史审计区映射

以下内容进入历史审计区：

- 所有每日作战输入包。
- 未被允许参与今日判断的输入。
- 判断时使用过的证据说明。
- 被排除的输入及排除原因。

历史审计区不能反向抢占当前行动指令。

## 4. 输入不足时的降级规则

如果输入不足：

- 没有 `key_event` 且没有 `current_question`：不得生成强判断。
- 没有 `constraints_or_forbidden`：必须提醒当前禁止事项缺失。
- `use_for_today_judgment = unclear`：不得生成今日唯一动作。
- 多个动作冲突：必须先要求澄清，不输出并列待办。

## 5. 与长期记忆的边界

W7 输入不自动进入长期记忆。

在 M0 完成前，日常输入只能用于：

- 今日判断。
- 历史审计。
- 阶段复盘。

不得自动升级为长期偏好、长期事实或个人模型。
