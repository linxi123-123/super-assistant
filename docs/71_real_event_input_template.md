# 真实事件输入模板

## 1. 用户每次输入真实事件时需要填写

1. 今天真实发生了什么？
2. 我现在最担心什么？
3. 我现在最想做什么？
4. 我现在不确定什么？
5. 我今天实际推进了什么？
6. 我有没有偏离当前阶段目标？
7. 我希望军师提醒我什么？
8. 有哪些事情我现在不该做？

## 2. 填写原则

- 优先写真实发生的事情，不写抽象愿望。
- 优先写当前困惑，不写长期空泛目标。
- 优先写实际推进，不写计划清单。
- 必须写“现在不该做什么”。
- 必须允许军师指出反方判断。

## 3. 建议真实事件模板

### 示例 1

我今天看了个人作战室页面，但感觉不够直观，更像列表，不像作战室。

### 示例 2

我担心现在的数据都是测试 case，不是真实事件，所以无法判断系统是否真的有价值。

### 示例 3

我觉得页面设计不够明确，第一屏没有强烈告诉我现在局势和最重要动作。

### 示例 4

我不想继续在假数据上优化页面，否则可能浪费时间。

### 示例 5

我现在真正想知道的是：这个超级军师项目下一步到底应该继续怎么走，还是应该暂停修正方向。
