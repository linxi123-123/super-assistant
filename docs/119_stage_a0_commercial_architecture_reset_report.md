# A0 阶段报告：商业级个人超级军师总体落地方案与工程路线重设

生成时间：2026-06-11

## 1. 本阶段为什么重启路线

用户明确指出：当前项目不能继续停留在本地静态页面、JSON、验收文档和作战室原型。

系统需要成为真正有商业价值的个人超级助理 / 个人超级军师。

## 2. 用户最新需求是什么

用户要求：

- 部署在公司服务器上。
- 可以接入大模型。
- 可以联网获取真实外部信息。
- 覆盖个人生活、工作、项目管理、市场/金融等多类信息。
- 隐私是红线。
- 敏感信息加密保存在本地或公司服务器。
- 不能把个人隐私、持仓、资产、关系、公司内部信息直接泄露给外部模型。
- 系统要能提出具体分析和行动建议。

## 3. 旧路线为什么不足

旧路线不足在于：

- 过度做页面、文档、快照、审计。
- 缺少真实外部数据。
- 缺少大模型推理能力。
- 缺少用户真实长期记忆。
- 缺少用户持仓、关注、项目、生活、工作上下文。
- 系统看起来安全，但真实使用价值不足。

## 4. 新总体架构是什么

新架构是：

```text
隐私优先的联网大模型个人超级军师系统
```

六层包括：

- Private Vault
- External Intelligence
- Privacy Governance
- LLM Reasoning
- Local Advisor Judge
- Action Layer

## 5. 隐私红线是什么

真实姓名、手机号、邮箱、身份证、资产金额、持仓数量、成本价、账户信息、公司内部机密、私人关系、健康信息、家庭敏感信息、详细地理位置、合同/财务/法律敏感内容默认不得直接发送给外部大模型。

## 6. 长期记忆如何设计

长期记忆分为：

- Raw Event
- Candidate Memory
- Confirmed Memory
- User Model Hypothesis
- Advisor Rule

所有长期记忆必须具备来源、证据、置信度、敏感级别、用户确认、有效期、是否允许进入建议、是否允许进入 LLM 上下文、修订历史等字段。

## 7. 外部情报如何接入

外部情报后续接入市场、新闻、公告、宏观、网页搜索、邮件、日历、文档、企业内部系统等。

所有外部数据必须有来源、时间戳和可信度。

## 8. 大模型如何安全使用

所有模型调用必须经过 LLM Gateway。

模型只接收经过脱敏和最小化处理后的任务包，不直接访问私密数据库。

输出必须由本地军师判断层二次审查。

## 9. 真正 MVP 是什么

真正 MVP 是三个高价值闭环：

- 市场军师
- 项目军师
- 个人决策军师

## 10. 30 天路线是什么

30 天路线：

- 第 1-3 天：A0-A1 架构和隐私模型
- 第 4-7 天：A2 私密数据保险库最小实现
- 第 8-12 天：A3 LLM Gateway 最小实现
- 第 13-17 天：A4 市场军师 MVP
- 第 18-22 天：A5 项目军师 MVP
- 第 23-27 天：A6 个人决策军师 MVP
- 第 28-30 天：整合与验收

## 11. 下一阶段建议

建议进入 A1：隐私与数据分级模型。

A1 仍需要用户单独确认。

## 12. 是否触碰禁止事项

A0 未触碰：

- app 页面
- SQLite schema
- SQLite 写入
- 真实隐私数据
- 外部 API
- 大模型 API
- npm / 框架
- 开发实现

## 13. 结论

当前项目应从本地静态作战室原型，转向隐私优先的联网大模型个人超级军师系统。是否进入 A1，需要用户单独确认。
