# W6 第一屏当前语境根因与修正目标

生成时间：2026-06-11

阶段：W6：第一屏当前语境与阶段时效规格修正

## 2. 当前缺陷

### A. 当前阶段不清楚

页面虽然显示局势和动作，但用户不一定知道当前内容属于哪个阶段语境。

用户可能会混淆：

- 这是 W5B 的真实事件快照判断？
- 这是 W5D 的页面改造结果？
- 这是 W6 当前阶段判断？
- 这是历史快照留下来的判断？

根因是第一屏缺少一个明确、优先级足够高的当前阶段标签。页面目前更像展示“当前作战室内容”，但没有先声明“当前项目阶段是什么、这条判断处在哪个验收语境里”。

### B. 判断时效不清楚

页面没有足够明确：

- 判断生成时间。
- 判断适用时间窗口。
- 判断是否仍然有效。
- 什么时候需要重新生成判断。

这会导致用户把一个阶段性快照误当成长期稳定判断。对于个人军师系统来说，判断时效必须可见，否则旧判断会干扰新决策。

### C. 当前动作 vs 历史动作混淆

页面里同时存在：

- `today_action`
- `recent_history`
- 历史测试模块
- 历史快照审计区

这些内容如果没有明确分区，用户容易不清楚：

- 哪个动作是现在要做的。
- 哪些动作只是历史记录。
- 哪些内容只是审计，不是当前建议。

根因不是模块数量过多，而是“当前行动指令”和“历史审计材料”之间的边界没有显式声明。

### D. 当前项目阶段与长期架构关系不清楚

项目已经记录“长期个人军师：记忆治理与判断闭环”架构图，并新增了长期记忆治理差距分析。

但当前项目仍处在第一屏 / 真实事件作战室阶段，尚未进入长期记忆治理中心实现。

如果第一屏不显示当前阶段和禁止事项，用户可能误以为：

- 已经进入 M0 记忆治理中心开发。
- 长期架构图已经是当前功能。
- 系统已经具备长期记忆治理能力。

这会让产品验收和开发边界混乱。

## 3. 修正目标

W6 要让第一屏规格明确显示：

- 当前阶段。
- 当前判断生成时间。
- 当前判断有效期 / 适用窗口。
- 当前唯一动作。
- 历史动作 / 历史模块只是审计。
- 当前不应进入哪些阶段。
- 下一阶段需要用户确认。

W6 不开发页面，只定义规格。是否进入 W6D 页面最小改造，需要用户单独确认。
