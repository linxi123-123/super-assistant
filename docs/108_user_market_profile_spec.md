# 用户市场画像规格

生成时间：2026-06-11

## 1. 目标

用户市场画像用于让市场军师知道：

- 用户关注哪些标的。
- 用户持有哪些标的。
- 用户关注或持有的原因。
- 用户风险偏好和禁止建议边界。

S0 阶段只做本地手动维护，不接券商账户，不猜测用户持仓。

## 2. 数据结构

```json
{
  "watchlist": [
    {
      "symbol": "",
      "name": "",
      "market": "A股 / 港股 / 美股",
      "why_follow": "",
      "risk_notes": ""
    }
  ],
  "holdings": [
    {
      "symbol": "",
      "name": "",
      "market": "",
      "position_direction": "持有 / 观察 / 已卖出",
      "cost_basis_optional": "",
      "position_size_optional": "",
      "buy_reason": "",
      "risk_tolerance": "",
      "notes": ""
    }
  ],
  "market_preferences": {
    "preferred_markets": [],
    "style": "短线 / 中线 / 长线 / 观察",
    "risk_tolerance": "",
    "forbidden_advice": [
      "不直接给买卖指令",
      "不根据单日波动下结论"
    ]
  }
}
```

## 3. 字段说明

### `watchlist`

用户关注标的列表。

用途：

- 当用户问“今天股市怎么样”时，系统可以主动检查关注标的是否受影响。

注意：

- 关注不等于持有。
- 系统不能把 watchlist 里的股票当成用户持仓。

### `holdings`

用户手动确认的持仓列表。

要求：

- 所有持仓必须用户手动确认。
- 系统不能猜测用户持仓。
- 成本和仓位可以为空。
- 不要求用户输入敏感金额。

### `market_preferences`

用户市场偏好和建议边界。

必须包含：

- 偏好市场。
- 投资/观察风格。
- 风险承受能力。
- 禁止建议。

## 4. 安全和隐私要求

- 不接券商账户。
- 不要求用户输入真实资金规模。
- 不自动推断持仓。
- 不上传画像。
- 不外传画像。
- 不直接给买卖指令。
- 不根据单日波动下结论。

## 5. S0 使用方式

S0 只允许读取或参考本地手动维护的画像。

如果画像为空，市场军师应该提示：

```text
我还不知道你关注和持有哪些标的。你可以先维护 watchlist / holdings。
```
