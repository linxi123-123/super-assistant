# 个人作战室 V1 PRD

## 1. 产品定位

个人作战室 V1 是超级军师系统的主界面。

它不是：

- 聊天页
- 数据库后台
- 历史审计页
- 普通 dashboard
- 待办列表
- 新闻摘要

它是：

- 当前局势面板
- 军师判断入口
- 风险与机会雷达的本地雏形
- 承诺与行动的收敛界面
- 历史判断和当前状态之间的桥梁

个人作战室的核心价值不是展示更多信息，而是帮助用户在复杂状态中看到“现在最该判断什么、做什么、不做什么”。

## 2. V1 核心问题

个人作战室 V1 要回答：

- 我现在处于什么阶段？
- 我当前最重要的目标是什么？
- 当前最重要的项目是什么？
- 最近出现了哪些高价值信号？
- 哪些风险正在变大？
- 哪些承诺没有完成？
- 哪些模型假设正在变化？
- 今天最应该做什么？
- 哪些事情现在不该做？
- 系统为什么这么判断？

## 3. V1 用户价值

### 如何避免变成普通看板

普通看板展示数据，个人作战室展示判断。

V1 的第一屏必须围绕阶段、信号、风险、承诺、今日动作组织，而不是围绕表格、统计图或消息流组织。

### 如何体现军师感

军师感来自六件事：

- 明确当前局势
- 抓住高价值信号
- 给出推荐动作
- 同时给出反方判断
- 展示证据链
- 追踪反馈和模型修正

如果只有结论，没有证据和反方判断，就不是军师。

### 如何帮助用户收敛注意力

V1 必须限制“今日最重要动作”的数量。默认只展示一个最重要动作，最多展示少量候选，但必须明确主动作。

作战室不鼓励用户扩功能、开新坑或追热点。它应该优先提醒用户完成当前阶段门槛。

### 如何让用户看到当前局势

当前局势由以下信息组合而成：

- 当前阶段
- 当前目标
- 当前主要项目
- 当前风险
- 当前机会
- 当前承诺
- 最近高价值信号
- 最近模型修正

这些信息必须在页面第一屏形成一张“局势图”，而不是散落在多个页面。

### 如何从历史审计面板升级为当前作战面板

历史审计面板回答：“过去这条判断是否可靠？”

个人作战室回答：“基于过去的判断和当前阶段，我现在应该注意什么？”

V1 应复用历史审计面板的数据和证据链，但展示目的要从复核历史转向指导当前行动。

## 4. V1 不做什么

V1 明确不做：

- 聊天机器人
- 外部情报
- 自动执行
- 多用户
- 移动 App
- 新数据源
- 自动发消息
- 自动操作电脑
- 复杂项目管理
- 复杂 UI 美化

## 5. 成功标准

个人作战室 V1 成功，不是因为页面好看，而是因为用户打开后能在 30 秒内知道：

- 当前局势
- 当前风险
- 当前最重要动作
- 哪个判断最需要复盘
- 哪个承诺最需要处理

V1 通过的最低标准是：用户不需要打开聊天框解释背景，也能理解系统为什么给出当前提醒。
