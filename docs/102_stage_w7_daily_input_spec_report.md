# W7 阶段报告：真实作战室日常输入机制规格设计

生成时间：2026-06-11

## 1. 本阶段目标

W7 的目标是定义真实作战室的日常输入机制，让系统从一次性真实事件快照，走向可持续的每日输入与当前判断生成机制。

## 3. 核心规格结论

W7 建议采用“每日作战输入包”。

用户每天最少需要提供：

- 今日真实事件，或
- 当前最想判断的问题。

同时建议记录：

- 项目状态变化。
- 今日承诺动作。
- 当前限制和禁止事项。
- 证据说明。
- 敏感度。
- 是否允许参与今日判断。

## 4. 如何服务第一屏

每日作战输入包映射到第一屏五个关键问题：

- 当前局势一句话。
- 当前最大矛盾。
- 军师直接判断。
- 今日唯一动作。
- 不要做清单。

如果输入不足，系统应降级为待澄清，而不是强行输出判断。

## 5. 与历史审计的关系

所有日常输入都可以进入历史审计。

但只有被允许参与今日判断的输入，才能进入当前作战室判断。

历史审计不得抢占当前唯一动作。

## 6. 与长期记忆治理的关系

W7 不自动沉淀长期记忆。

在 M0 完成前，日常输入只能用于：

- 今日判断。
- 历史审计。
- 阶段复盘。

不得自动升级为长期偏好、长期事实或个人模型。

## 7. 是否触碰禁止事项

本阶段未触碰：

- app 页面。
- JSON 数据。
- SQLite schema。
- SQLite 写入。
- 新数据源。
- 外部网络。
- 外部情报。
- 执行代理。
- M0 记忆治理中心开发。
- npm / 框架 / 第三方库。
- 后端服务。

## 8. 报告结论

W7 已完成日常输入机制规格设计。是否进入 W7D 最小实现，或转入 M0 长期记忆治理中心规格设计，需要用户单独确认。
