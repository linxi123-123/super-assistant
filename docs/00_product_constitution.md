# 产品宪法

## 一句话定义

个人超级军师系统，是一个持续运行的个人局势智能体。它通过多源感知理解用户真实处境，通过长期记忆建立动态个人模型，通过外部情报雷达识别机会和风险，通过推理系统做战略判断，并在关键时刻主动触达用户，最终逐步具备跨工具执行能力。

更短一点：

> 它不是 AI 聊天助手，而是个人局势系统。

## 产品本质

普通 AI 助手的主循环是：

```text
用户输入 -> AI 回答 -> 结束
```

超级军师的主循环是：

```text
持续感知
-> 形成事件
-> 更新记忆
-> 更新个人模型
-> 扫描外部世界
-> 识别机会 / 风险 / 偏离 / 承诺
-> 判断是否值得主动触达
-> 给出军师级建议
-> 必要时执行任务
-> 追踪结果
-> 修正模型
```

## 12 条原则

1. 它不是聊天机器人。聊天只是入口之一，系统核心是持续运行的局势判断。
2. 它不是日记工具。用户输入只是数据来源之一，重点是转成事件、记忆、判断和行动。
3. 它不是新闻摘要工具。外部信息必须和用户目标、处境、风险、机会发生关系。
4. 它不是待办工具。任务必须服务于长期目标和当前局势。
5. 它不优化“让用户舒服”，而是提高长期目标达成概率。
6. 它必须主动。如果永远等用户问，它就不是军师。
7. 它必须有证据链。所有判断都要说明证据、置信度、反例和验证方式。
8. 它必须区分事实、推测、建议和价值判断。
9. 它必须记住用户的判断，而不只是记住用户说过的话。
10. 它必须有主动触达机制，但触达必须克制。
11. 它必须有执行权限边界。
12. MVP 可以小，但主循环不能错。

## 不可妥协项

- 主循环必须是感知 -> 信号识别 -> 主动触达。
- 记忆必须结构化，不是聊天记录堆积。
- 判断必须有证据链和置信度。
- 系统必须能反驳用户。
- 外部情报必须和用户目标相关。
- 执行动作必须绑定目标和结果追踪。
- 首页不能以聊天框为中心。
- 必须有反馈进化机制。
- 必须有权限和记忆审计。

## 可妥协项

- V1 不接入全部数据源。
- V1 不全天候监听。
- V1 不做手机 App。
- V1 不做复杂自动化。
- V1 不做全自动执行。
- V1 外部情报源可以少。
- V1 界面可以简单。
- V1 模型可以调用现成 API。
