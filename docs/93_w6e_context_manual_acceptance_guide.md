# W6E 第一屏当前语境人工验收指南

生成时间：2026-06-11

阶段：W6E：第一屏当前语境人工验收

## 1. 本阶段目标

W6E 的目标是人工判断 W6D 第一屏是否解决了 W5E 发现的问题。

需要重点判断：

- 当前阶段是否清楚。
- 判断时效是否清楚。
- 当前动作和历史动作是否清楚区分。
- 历史模块是否被明确标注为审计。
- 用户是否不会误以为可以进入 M0 / 外部情报 / 执行代理。

W6E 不是开发阶段，不改页面、不改 JSON、不改 SQLite、不接新数据源、不进入 M0、外部情报或执行代理。

## 2. 用户打开地址

```text
http://127.0.0.1:8766/app/war_room.html
```

请用户在浏览器中打开或刷新该页面后进行人工验收。

## 3. 人工验收原则

用户不要只看页面是否能加载。

必须判断：

- 第一屏是否能让我知道“我现在处于哪个阶段”。
- 第一屏是否能让我知道“这个判断什么时候生成，什么时候有效”。
- 第一屏是否能让我知道“现在唯一动作是什么”。
- 第一屏是否能让我知道“哪些内容只是历史审计”。
- 第一屏是否能防止我跳到 M0、外部情报或执行代理。

## 4. 验收重点

### 当前阶段

用户应能在 10 秒内看到当前阶段，并理解当前仍处于作战室第一屏验收链路，而不是长期记忆治理实现阶段。

### 判断时效

用户应能看到判断生成时间和判断有效期 / 适用窗口，并理解当前判断不是永久战略判断。

### 当前动作 vs 历史动作

用户应能明确区分：

- 当前唯一动作：现在真正要做的事情。
- 历史模块：用于审计和追溯，不是当前行动指令。

### 禁止事项

用户应能清楚看到当前不得进入：

- 外部情报。
- 执行代理。
- 新数据源。
- M0 记忆治理中心开发。
- 跳过 W6E 人工验收。

### 下一阶段门槛

用户应能知道下一阶段需要单独确认，不会被页面暗示自动进入。

## 5. 验收后动作

请用户填写：

```text
docs/95_w6e_context_manual_acceptance_draft.md
```

在人工评分完成前，不建议进入下一阶段。
