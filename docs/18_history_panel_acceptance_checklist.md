# 历史面板人工验收清单

人工打开：

```text
http://127.0.0.1:8766/app/history.html
```

## 1. 总体验收问题

逐项检查：

- [ ] 页面是否能正常加载。
- [ ] 是否明确显示这是“历史审计面板”。
- [ ] 是否能看到 summary。
- [ ] 是否能看到 15 条 cases。
- [ ] 是否能点击 case。
- [ ] 是否能看到完整链路。
- [ ] 是否能看到 signal distribution。
- [ ] 是否能看到 hypothesis distribution。
- [ ] 是否能看到 confidence changes。
- [ ] 是否能看到 audit flags。
- [ ] 是否没有不可接受的横向溢出。
- [ ] 移动端宽度是否仍可阅读。

## 2. 每条 case 的人工审查问题

每条 case 都要检查：

- [ ] 输入事件是否被正确理解。
- [ ] candidate_memory 是否值得保存。
- [ ] hypothesis 是否是有价值的个人模型假设。
- [ ] signal_type 是否准确。
- [ ] touchpoint 是否真的有军师提醒价值。
- [ ] counter_argument 是否具体，不是模板化反驳。
- [ ] recommended_action 是否明确。
- [ ] consequence_if_ignored 是否具体。
- [ ] feedback 是否合理。
- [ ] outcome 是否合理。
- [ ] model_revision 是否合理。
- [ ] confidence change 是否符合反馈方向。
- [ ] 是否能从页面看出证据链。
- [ ] 是否存在复述输入但没有新判断。
- [ ] 是否存在伪反迎合。
- [ ] 是否存在高价值信号沉默。
- [ ] 是否存在 scope creep 漏判。

## 3. 评分标准

每条 case 人工评分 0-5：

- 链路完整性：_/5
- 判断可解释性：_/5
- 军师提醒价值：_/5
- 反迎合质量：_/5
- 模型修正合理性：_/5
- 页面可审计性：_/5

总分：_/30

建议门槛：

- 单条 >= 24/30：通过
- 18-23：需要优化
- < 18：必须修正

## 4. 进入下一阶段门槛

只有满足以下条件，才建议进入“个人作战室 V1”：

- [ ] 15 条 case 人工平均分 >= 24/30。
- [ ] 没有严重伪反迎合。
- [ ] 没有高价值信号沉默。
- [ ] 没有明显无法解释的模型修正。
- [ ] 页面能帮助用户看懂判断链路。
- [ ] 用户确认历史面板有审计价值。

在人工完成 15 条 case 审查前，不建议进入个人作战室 V1。
