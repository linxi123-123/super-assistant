# W5E 第一屏规格修正建议

## 1. 修正目标

W5E 人工验收结论是：第一屏结构通过，但规格仍需补一层“当前阶段语境”。

当前页面能够回答五个关键问题，但其中部分回答来自 W5B/W5D 时点的真实快照。进入 W5E 后，用户正在做人工验收，因此“今日唯一动作”和“军师直接判断”需要区分：

- 当时快照判断
- 当前阶段判断
- 下一步阶段门槛

本文件只记录规格修正建议，不修改页面、不修改 JSON、不写数据库。

## 2. 已发现规格缺陷

### 缺陷 1：第一屏缺少当前阶段标识

现象：

页面展示真实快照内容，但没有明确告诉用户这份判断对应哪个阶段、哪个时点、是否仍是当前动作。

风险：

如果阶段继续推进，第一屏会保留旧判断。用户可能看到“完成真实事件快照生成与校验”，但当前实际已经进入 W5E 人工验收。

修正建议：

后续规格中增加：

- `stage_context.current_stage`
- `stage_context.snapshot_stage`
- `stage_context.generated_at`
- `stage_context.valid_for_stage`
- `stage_context.staleness_warning`

### 缺陷 2：今日唯一动作没有区分历史动作和当前动作

现象：

当前 `today_action.only_action` 是：

`完成真实事件快照生成与校验，确认它能回答五个关键问题。`

这对 W5B 是正确的，但对 W5E 已经不够准确。

风险：

“今日唯一动作”如果不能随阶段更新，会削弱作战室的现场感，最终退化为静态报告阅读器。

修正建议：

后续规格中将行动拆为：

- `snapshot_action.original_only_action`
- `current_action.only_action`
- `current_action.done_definition`
- `current_action.blocked_actions`
- `current_action.requires_user_confirmation`

### 缺陷 3：军师直接判断缺少时效标签

现象：

当前军师判断仍在说“不应该继续加模块，应该先重建快照和第一屏判断层级”。但 W5D 已经完成了第一屏最小改造。

风险：

如果没有时效标签，用户无法判断这是“当时正确的判断”还是“现在仍然有效的判断”。

修正建议：

后续规格中增加：

- `advisor_brief.judgment_time_scope`
- `advisor_brief.still_valid`
- `advisor_brief.changed_since_snapshot`
- `advisor_brief.current_judgment`

### 缺陷 4：解释层略重，首屏压缩仍可加强

现象：

第一屏已经优先展示判断，但 `why / counter / consequence` 信息仍较多。

风险：

用户打开页面时可能仍需要读较多文字，影响“30 秒内知道现在该干什么”的作战室目标。

修正建议：

后续规格中将首屏解释分层：

- 第一层：局势、最大矛盾、直接判断、唯一动作、不要做
- 第二层：为什么重要、反方判断、忽略后果
- 第三层：证据链、审计字段、来源元数据

### 缺陷 5：旧模块次级区完整，但与第一屏关系需要更明确

现象：

旧 9 个模块被保留在下方，并且导航、折叠可用。但用户仍可能不清楚这些模块是“证据与审计支撑”，不是第一屏主判断。

风险：

如果旧模块重新抢占注意力，页面会回退成模块列表/dashboard。

修正建议：

后续规格中明确旧模块定位：

- `secondary_modules.role = evidence_and_audit_support`
- 旧模块默认不参与第一屏主行动判断
- 旧模块只用于解释判断来源、查看细节、审计链路

## 3. W6 建议范围

建议进入 W6：第一屏当前语境与阶段时效规格修正。

W6 只允许：

- 设计当前阶段语境字段
- 设计当前动作和历史动作的区分方式
- 设计军师判断时效标签
- 设计首屏三层信息压缩规则
- 设计旧模块与第一屏判断的证据关系
- 输出规格文档和验收标准

W6 继续禁止：

- 修改 JSON
- 修改 SQLite
- 写数据库
- 接新数据源
- 读取外部网络
- 引入 npm / 框架
- 进入外部情报
- 进入执行代理
- 开发新页面
- 做 UI 美化

## 4. 进入 W6 的门槛

建议门槛：

- 用户确认 W5E 验收通过
- 用户确认 W6 只做规格修正，不做页面实现
- 用户确认 W6 重点是“当前语境/阶段时效”，不是外部情报或执行代理

## 5. W5E 结论

W5E 通过。

第一屏已经具备作战室简报形态，但如果不补当前阶段语境，后续会出现一个严重退化风险：页面看起来像军师判断，实际展示的是上一阶段快照。

因此，下一步不应直接扩功能，而应先把“当前判断”和“历史快照判断”的规格边界定清楚。
