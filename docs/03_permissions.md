# 权限与隐私边界

## 设计原则

这个系统越懂用户，越必须克制。权限、审计、删除机制必须从第一版就存在。

用户必须知道：

- 系统知道什么。
- 为什么知道。
- 从哪里知道。
- 是否正在使用。
- 能否删除。
- 是否允许用于建议。

## 数据权限分级

| 等级 | 数据类型 | 处理方式 |
| --- | --- | --- |
| P0 | 普通信息 | 可自动处理 |
| P1 | 工作信息 | 可处理但需审计 |
| P2 | 私人关系 | 默认不进入长期记忆，需确认 |
| P3 | 财务 / 健康 / 位置 | 高敏感，需单独授权 |
| P4 | 密码 / 证件 / 核心隐私 | 禁止自动使用 |

## 执行权限分级

| 等级 | 操作 |
| --- | --- |
| E0 | 只建议 |
| E1 | 生成草稿 |
| E2 | 整理本地资料 |
| E3 | 调用外部工具但需确认 |
| E4 | 低风险自动执行 |
| E5 | 高风险永远确认 |

## V1 权限策略

- 默认只启用 P0-P1 数据。
- P2 以上数据默认不进入长期记忆。
- V1 执行权限最高到 E1。
- 所有长期记忆都必须可审计、可修改、可删除。
- 所有建议必须展示引用了哪些记忆或事件。

## 必须实现的基础能力

- 记忆审计。
- 一键删除记忆。
- 是否允许用于建议的开关。
- 敏感度字段。
- 用户确认字段。
