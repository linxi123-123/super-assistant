# 个人局势军师 V1 PRD

## V1 定位

个人局势军师 V1 是超级军师终局系统的最小可运行切片。

它不追求完整 Jarvis，也不追求全自动感知，但必须从第一天起保留军师主循环：

```text
事件输入 -> 局势快照 -> 候选记忆 -> 个人模型假设 -> 信号识别 -> 主动提醒 -> 用户反馈 -> 模型修正
```

## V1 目标

让系统能够回答：

- 用户现在的核心目标是什么？
- 用户正在推进什么项目？
- 用户最近做了什么重要判断？
- 用户有哪些承诺未完成？
- 用户当前主要风险是什么？
- 哪些记忆影响了当前建议？
- 现在是否应该主动提醒用户？

## V1 核心功能

### 1. 个人档案

记录用户基础背景、当前阶段、能力结构、资源结构、价值观、风险偏好、沟通偏好和隐私偏好。

验收标准：

- 能展示当前个人档案。
- 每个关键字段支持来源说明。
- 支持手动更新。

### 2. 目标管理

记录长期目标、中期目标、短期目标，以及成功标准、失败标准和当前状态。

验收标准：

- 每个目标有时间范围、重要性、状态和关联项目。
- 系统建议必须能关联至少一个目标。

### 3. 项目管理

记录正在推进的项目、阶段、进展、关键风险、下一步动作和相关文件。

验收标准：

- 能看到当前项目列表。
- 每个项目能关联目标、事件、记忆、决策和行动。

### 4. 每日事件输入

用户每天输入或系统记录关键事件。

事件类型包括：

- 行为事件
- 判断事件
- 承诺事件
- 情绪/状态事件
- 关系事件
- 项目进展事件
- 外部情报事件
- 风险事件
- 机会事件
- 偏离事件

验收标准：

- 输入内容不会直接进入长期记忆，而是先形成事件。
- 事件包含来源、时间、重要性、敏感度、置信度和原始证据。

### 5. 每日局势快照

系统每天根据事件生成局势快照。

输出包括：

- 今日关键事件
- 当前状态判断
- 目标推进情况
- 承诺变化
- 风险变化
- 候选记忆
- 候选信号
- 明日最重要动作

验收标准：

- 快照必须区分事实、推测、建议。
- 至少给出一条可行动建议。

### 6. 长期记忆库

从事件中提取候选记忆，经重要性、敏感度和置信度评估后保存。

验收标准：

- 记忆可按类型查看。
- 每条记忆有来源事件、证据、置信度、是否确认、是否过期。
- 支持确认、修改、删除。

### 7. 个人模型假设

系统维护对用户的阶段性理解，但以“假设”形式保存。

验收标准：

- 每条假设包含证据、反例、置信度和验证计划。
- 用户可以反馈“准确 / 不准确 / 需要修正”。

### 8. 信号识别

系统从事件、记忆、目标和个人模型中识别信号。

信号类型包括：

- 偏离信号
- 机会信号
- 风险信号
- 承诺信号
- 决策信号
- 状态信号
- 模式信号

验收标准：

- 每个信号有重要性、紧急性、目标相关度、可行动性、置信度、打扰成本。
- 不是所有信号都触达。

### 9. 主动军师提醒

系统根据高价值信号生成克制的主动提醒。

提醒格式：

```text
我为什么提醒你？
我观察到了什么？
这和你的哪个目标有关？
风险或机会是什么？
我建议你现在做什么？
如果不做，可能有什么后果？
```

验收标准：

- 提醒必须有证据。
- 提醒必须有建议动作。
- 支持用户反馈提醒是否准确。

### 10. 反方挑战模式

当用户做重大判断或计划时，系统必须给出反方观点。

验收标准：

- 不能只赞同用户。
- 必须指出至少一个风险、一个反例、一个需要验证的假设。

## V1 暂不做

- 全天候麦克风监听。
- 全自动读取所有聊天软件。
- 自动发邮件、自动购买、自动转账。
- 手机 App。
- 复杂多 Agent 编排。
- 高风险自动执行。

## V1 首页

首页是个人作战室，而不是聊天框。

首屏模块：

- 当前阶段判断
- 当前核心目标
- 正在推进的项目
- 当前主要风险
- 当前主要机会
- 近期承诺
- 偏离信号
- 今日最重要动作
- 需要确认的记忆
- 军师主动建议
