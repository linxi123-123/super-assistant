# COMMERCIAL-V1 产品规格：商业级个人超级军师可用系统

## 定位

COMMERCIAL-V1 的目标不是继续做静态作战室或文档型原型，而是把系统升级为一个可真实使用的个人超级军师最小商业闭环。

用户应该能够直接说：

- 今天股市怎么样
- 腾讯今天怎么样
- 我这个项目下一步该怎么做
- 我纠结要不要继续做这个产品
- 我今天很烦，不知道该先做什么

系统需要自动判断问题类型，结合可用外部证据、长期记忆摘要和本地安全审查，输出有边界、有来源、有行动建议、可反馈纠错的回答。

## 核心用户价值

1. 不再要求用户手动选择事件类型。
2. 不把无来源信息伪装成实时事实。
3. 不给买入、卖出等直接交易指令。
4. 能显示本次回答用了哪些外部信息和记忆状态。
5. 能对回答质量进行评分，低质量回答自动降级。
6. 能记录用户反馈，把错误回答和错误记忆纳入复审。
7. 能长期使用，但长期记忆不会无治理地污染后续判断。

## 当前能力

- 统一军师入口：`POST /api/advisor/chat`
- 四类 advisor 路由：`market_advisor` / `project_advisor` / `decision_advisor` / `general_advisor`
- 可选真实 LLM provider：DeepSeek 优先，兼容 Kimi/GPT，mock fallback
- 外部情报 MVP：OpenWeather、Open-Meteo、Tavily、Finnhub
- 来源可信度、时效性、证据包和冲突检测
- 回答质量评分与降级
- 本地长期记忆摘要、候选记忆和历史查询
- 记忆写入策略：只有评分通过且未降级的回答允许产生候选记忆
- 用户反馈 API：回答反馈和记忆反馈
- 记忆健康 API：候选、确认、过期、冲突、隔离、待复审统计
- 前端主入口显示 provider/model/llm_mode/local_judge_status、来源、记忆、评分和反馈按钮

## 非目标

COMMERCIAL-V1 不做以下事情：

- 不接券商账户、银行账户、邮件、日历、联系人或公司内部系统
- 不自动交易、不自动执行电脑操作、不启用执行 agent
- 不把真实 API key 打印、写入文档或提交
- 不把无来源、低可信、过期信息当事实
- 不做向量数据库、不做复杂个人画像系统
- 不引入 npm、React、Vue、Next 或其他前端框架

## 验收口径

COMMERCIAL-V1 合格的标准是：

- 用户能直接对话，而不是先填事件表单
- 市场、项目、决策、普通问题可以稳定分流
- 回答必须可审计、可降级、可反馈
- 记忆必须受治理，不能把低质量回答直接沉淀为长期判断依据
- 前端能让普通用户看见系统边界和反馈入口
