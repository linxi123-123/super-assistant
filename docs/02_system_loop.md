# 主循环设计

## 终局主循环

```text
多源输入
-> 原始数据
-> 事件抽取
-> 事件分类
-> 每日局势摘要
-> 候选记忆提取
-> 长期记忆保存
-> 个人模型更新
-> 外部情报扫描
-> 信号识别
-> 重要性评分
-> 主动触达
-> 用户反馈 / 系统执行
-> 结果追踪
-> 模型修正
```

## V1 最小主循环

```text
手动事件输入
-> 事件结构化
-> 每日局势快照
-> 候选记忆提取
-> 个人模型假设更新
-> 信号识别
-> 主动提醒生成
-> 用户反馈
-> 记忆/模型修正
```

## 输入层级

```text
L0：完全手动输入
L1：每日复盘输入
L2：日历、任务、文档同步
L3：浏览器、邮件、消息摘要
L4：会议/语音场景授权记录
L5：本地屏幕/应用行为感知
L6：长期本地化环境感知
```

V1 默认支持 L0 和 L1，架构预留 L2-L6。

## 事件晋升为记忆

```text
原始输入
-> 事件
-> 每日摘要
-> 候选记忆
-> 重要性判断
-> 敏感度判断
-> 用户确认或自动保存
-> 长期记忆
-> 个人模型假设更新
```

## 触达决策

系统发现信号后，先评分再决定是否提醒。

评分维度：

- 重要性
- 紧急性
- 长期目标相关度
- 可行动性
- 置信度
- 误报风险
- 打扰成本

触达条件：

- 和核心目标强相关。
- 有明确行动价值。
- 证据链足够。
- 打扰成本低于潜在收益。
- 最近没有重复提醒同一问题。
