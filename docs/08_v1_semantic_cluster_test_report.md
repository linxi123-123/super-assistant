## 原 5 条是否明显改善

是。

原 5 条从第二轮的 25.8/30 提升到 27.3/30。

关键改善：

- 提醒统一成六段式。
- 触达不再复述输入，而是说明阶段关系和不行动后果。
- 承诺类反馈现在修正 `hyp_commitment_tracking`，不是错误修正第一个假设。

## 变体 5 条是否识别语义

是。

第二轮失败的 3 个变体已通过：

- “换一种说法输出” -> `judgment_quality_risk`
- “系统级入口被 OpenAI / Google / Apple 掌握” -> `platform_competition_risk`
- “想把语音/浏览器/邮件/健康数据都加进去，但判断质量未验证” -> `scope_creep_risk`

这说明当前已经不只是命中原句关键词，而是能覆盖一组语义线索。

## 仍然机械的地方

虽然通过当前测试，但仍有边界：

1. 语义簇仍是显式规则，不是模型理解。
2. negative examples 目前主要用于文档和简单惩罚，测试覆盖不足。
3. 多意图事件只取最高分 cluster，可能漏掉次要信号。
4. 进展信号仍是单独规则，不在 semanticClusters 中。
5. 记忆提取仍偏逐字保存，没有生成压缩后的抽象记忆。
