# 阶段 Q：个人作战室 V1 规格设计启动

## 1. 阶段定位

阶段 Q 是规格设计阶段，不是开发阶段。

目标是把已经通过验收的“超级军师主循环”升级为一个可设计、可验收、可开发的“个人作战室 V1”产品规格。

个人作战室 V1 不应退化成聊天页，也不应变成普通 dashboard。它应该成为用户每天判断局势、识别信号、校准行动和追踪结果的核心界面。

## 2. V1 核心问题

个人作战室 V1 必须回答：

- 我现在处在什么阶段？
- 当前最重要的信号是什么？
- 哪些判断来自证据，哪些只是模型假设？
- 哪些提醒值得立刻行动？
- 哪些判断需要反证或延后验证？
- 我的长期目标是否正在被短期冲动带偏？
- 军师模型最近如何修正了对我的理解？

## 3. V1 设计目标

个人作战室 V1 的目标：

- 展示当前局势，而不是展示聊天记录
- 突出高价值信号，而不是堆信息
- 显示证据链和模型假设，而不是只给结论
- 支持用户反馈和结果追踪，而不是一次性建议
- 支持反迎合提醒，而不是顺着用户扩张欲望
- 帮助用户做阶段决策，而不是制造更多任务

## 4. 初步模块范围

V1 可以包含以下模块，但需要在后续规格中继续收敛：

- 当前阶段面板
- 今日关键事件
- 高价值信号
- 军师提醒
- 个人模型假设
- 证据链审计
- 用户反馈
- 结果追踪
- 模型修正历史
- 下一步行动门槛

## 5. 明确不做

阶段 Q 不做：

- 真实页面开发
- UI 美化
- SQLite schema 修改
- 外部情报接入
- 邮件、浏览器、语音、位置、健康数据接入
- 自动执行代理
- 自动发送消息
- 自动操作电脑
- npm / 框架引入

这些能力只有在个人作战室 V1 规格通过后，才能进入独立阶段讨论。

## 7. 进入开发前门槛

只有满足以下条件，才建议进入个人作战室 V1 开发：

- PRD 明确不把系统做成普通聊天助手
- 信息架构能承载军师主循环
- 每个模块都有清晰输入、输出和验收标准
- 数据权限边界明确
- 不依赖尚未接入的新数据源
- 不要求执行代理先行
- 用户确认 V1 范围足够小，可以静态或本地最小实现

## 8. 当前结论

P3 已通过，可以进入阶段 Q。

但阶段 Q 的第一步应是规格设计，而不是直接开发。
