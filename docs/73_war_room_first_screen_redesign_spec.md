# 作战室第一屏重构方案

## 1. 当前第一屏问题

当前第一屏问题：

- 信息像列表
- 模块太平均
- 缺少压倒性的主判断
- 缺少“唯一动作”
- 缺少“不要做什么”
- 测试数据让页面缺乏真实紧迫感

现有页面能展示模块，但不能强烈表达：

- 你现在到底处在什么局势？
- 你最该担心什么？
- 今天唯一动作是什么？
- 哪些事现在绝对不要做？

## 2. 新第一屏结构

第一屏必须变成 5 个强模块。

### 1. 局势一句话

示例：

你现在不是缺功能，而是在验证“作战室是否真的能帮助你判断局势”。

要求：

- 一句话说清当前状态。
- 不超过两行。
- 不能是模块摘要。

### 2. 当前最大矛盾

示例：

工程链路已经跑通，但真实事件价值尚未验证。

要求：

- 明确冲突。
- 明确为什么现在卡住。
- 不能写成普通风险列表。

### 3. 军师直接判断

示例：

现在不应该继续加模块，也不应该进入外部情报；应该用真实事件重建作战室快照。

要求：

- 直接给判断。
- 必须包含反迎合内容。
- 必须告诉用户现在不该做什么。

### 4. 今日唯一动作

示例：

用 5 条真实事件生成 `real_war_room_snapshot_v1`，并重新验收页面。

要求：

- 只能有一个主动作。
- 必须有完成定义。
- 必须能当天完成或推进。

### 5. 不要做清单

示例：

- 不要继续优化假数据页面
- 不要进外部情报
- 不要进执行代理
- 不要接新数据源
- 不要做 UI 美化

要求：

- 明确阻止走偏。
- 优先展示。
- 不要藏在审计区底部。

## 3. 新页面原则

- 第一屏不是信息列表，而是军师简报。
- 次级模块才展示 signals / hypotheses / history。
- 先判断，后证据。
- 先行动，后列表。
- 先告诉用户不该做什么。
- 数据必须来自真实事件，不再用测试 case 验证作战室价值。

## 4. W5A 后续路线

W5A 不改页面。

后续路线：

- W5B：生成真实事件快照。
- W5C：再设计页面重构。
- W5D：才允许最小页面改造。

进入任何后续阶段都需要用户单独确认。
