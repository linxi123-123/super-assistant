# W7 真实作战室日常输入机制 PRD

生成时间：2026-06-11

## 1. 背景

W6D/W6E 已经解决第一屏当前语境问题：用户能区分当前判断、历史审计、下一阶段门槛和禁止事项。

W7 要解决的新问题是：

```text
真实作战室如何从一次性真实事件快照，变成用户每天可持续输入、可审计生成当前判断的机制？
```

W7 不是页面开发，不接新数据源，不写数据库，不进入外部情报或执行代理。

## 2. 产品目标

设计一个足够轻的日常输入机制，让用户每天用最少负担提供真实上下文，并让系统能基于这些输入生成作战室当前判断。

机制必须服务第一屏五个关键问题：

1. 当前局势一句话是什么？
2. 当前最大矛盾是什么？
3. 军师直接判断是什么？
4. 今日唯一动作是什么？
5. 现在不该做什么？

## 3. 用户价值

用户每天不需要写长日记，也不需要和系统聊天很久。

用户只需要提供当天最关键的少量事实、困惑、动作和约束，系统就能形成一个可审计的“今日作战室输入包”。

## 4. 核心原则

- 最小输入：每天不超过 5 个核心字段。
- 事件优先：优先记录真实发生的事，不记录泛泛想法。
- 当前判断优先：输入服务当前作战室判断，不直接沉淀长期记忆。
- 可审计：每个判断能追溯到当天输入。
- 不聊天化：输入不是聊天记录堆积。
- 不长期化：在 M0 前，不自动升级为长期记忆。
- 不自动执行：只生成判断和建议，不执行任务。

## 5. 输入形态

W7 建议日常输入采用“每日作战输入包”。

每日作战输入包包含：

- 今日真实事件。
- 当前最大困惑。
- 当前目标 / 项目状态变化。
- 今日已做或承诺要做的动作。
- 当前限制和禁止事项。

用户可以每天只填 1 条，但系统必须能识别输入是否足以生成作战室判断。

## 6. 成功标准

W7 规格通过后，应能回答：

- 用户每天最少输入什么。
- 哪些输入进入当前判断。
- 哪些输入只进入历史审计。
- 输入如何生成第一屏五个关键问题。
- 如何避免日常输入变成聊天记录。
- 如何避免在没有 M0 前过度沉淀长期记忆。

## 7. 非目标

W7 不做：

- 页面开发。
- 表单开发。
- JSON 写入。
- SQLite 写入。
- 外部数据源接入。
- 外部情报雷达。
- 执行代理。
- M0 长期记忆治理中心开发。
- 聊天功能。
- 自动同步。
