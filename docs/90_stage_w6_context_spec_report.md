# W6 阶段报告：第一屏当前语境与阶段时效规格修正

生成时间：2026-06-11

## 1. 本阶段目标

W6 的目标是基于 W5E 人工验收结果，修正个人作战室第一屏的当前语境和阶段时效规格。

W6 只解决规格问题，不进入页面开发。

## 2. 输入依据

W6 输入依据：

- W5D：真实事件第一屏最小页面改造结果。
- W5E：人工验收结果，评分 `27/30`。
- `docs/83_w5e_manual_acceptance_scores.md`
- `docs/84_w5e_first_screen_spec_corrections.md`
- 长期记忆治理图及 `docs/85_long_term_memory_governance_gap_analysis.md`
- 当前 `tasks/MASTER_ROADMAP.md`

## 3. W5E 发现的问题

W5E 通过，但发现以下规格问题：

- 当前页面内容仍带 W5B/W5D 语境。
- 第一屏缺少当前阶段标签。
- 判断时效不够明确。
- 当前动作和历史动作容易混在一起。
- 用户需要明确知道：这是当前判断，还是历史快照留下来的判断。

## 4. 当前语境字段设计

W6 定义第一屏语境字段：

- `current_stage_label`
- `judgment_generated_at`
- `judgment_valid_window`
- `current_action_label`
- `historical_context_notice`
- `next_stage_gate`
- `forbidden_now`

字段规格见：`docs/87_w6_first_screen_context_field_spec.md`。

## 5. 信息层级修正方案

W6 建议第一屏顺序：

1. 当前阶段标签
2. 判断生成时间 / 判断有效期
3. 局势一句话
4. 军师直接判断
5. 今日唯一动作
6. 不要做清单
7. 当前最大矛盾
8. 下一阶段门槛
9. 历史审计提示

信息层级规格见：`docs/88_w6_first_screen_information_hierarchy_spec.md`。

## 6. 验收标准

W6 规格通过标准包括：

- 用户能在 10 秒内看到当前阶段。
- 用户能在 10 秒内看到判断生成时间或判断时效。
- 用户能明确知道当前唯一动作。
- 用户能明确知道哪些内容是历史审计。
- 用户不会误以为已经进入长期记忆治理实现。
- 用户不会误以为可以进入外部情报或执行代理。

人工评分总分 30，通过门槛为总分 `>= 25/30`，且当前动作 vs 历史动作区分、当前阶段清晰度、判断时效清晰度达到硬门槛。

验收标准见：`docs/89_w6_context_acceptance_criteria.md`。

## 7. W6D 页面最小改造边界

W6D 若由用户单独确认，只允许最小修改：

- `app/war_room.html`
- `app/war_room.js`
- `app/war_room.css`

W6D 仅允许只读展示当前语境字段、历史审计提示、下一阶段门槛和禁止事项。

W6D 不允许改 JSON、SQLite、接新数据源、引 npm/框架、进入外部情报或执行代理。

边界草案见：`tasks/w6d_first_screen_context_minimal_build_plan.md`。

## 8. 为什么不进入 M0

长期记忆治理图已经作为长期架构参考记录，但当前 W6 目标不是记忆治理中心规格设计。

当前问题是第一屏把当前判断、历史快照、阶段语境和动作边界混在一起。因此 W6 应先修正作战室第一屏的当前语境与阶段时效规格。

M0 仍可作为后续可选阶段，但不能由 W6 自动进入。

## 9. 为什么不进入外部情报/执行代理

当前阶段不进入外部情报，因为外部信息会放大噪声，并进一步混淆当前判断与历史审计。

当前阶段不进入执行代理，因为执行会把未完全澄清的当前动作与历史动作混淆风险放大成真实行动风险。

W6 的职责是先把“现在是什么阶段、现在判断是否有效、现在唯一动作是什么”说清楚。

## 10. 是否触碰禁止事项

W6 本阶段只输出规格文档、字段设计、验收标准、下一阶段边界和路线图更新。

本阶段未触碰：

- app 页面
- JSON 数据
- SQLite schema
- SQLite 写入
- 新数据源
- 外部网络
- 外部情报
- 执行代理
- npm / 框架 / 第三方库
- 后端服务
- M0 记忆治理中心开发

## 11. 报告结论

W6 只完成第一屏当前语境与阶段时效规格修正。是否进入 W6D 页面最小改造，需要用户单独确认。
