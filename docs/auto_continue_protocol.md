# 自动继续协议

## 1. 自动继续规则

如果当前阶段目标未达成，且下一步仍属于当前阶段内的补强、测试、报告，可以自动继续。

例如：

```text
规则补强
-> 测试
-> 报告
-> 路线图更新
```

可以自动完成一个闭环。

## 2. 跨阶段强制停止

只要下一步涉及以下事项，必须停止并等待用户确认：

- SQLite
- 新框架
- npm
- 新数据源
- 架构重构
- 自动执行权限
- 权限敏感数据接入
- 产品化主动触达

## 3. 最多自动继续 5 个闭环

每个闭环为：

```text
规则修改
-> 测试
-> 报告
-> 路线图更新
```

达到 5 个闭环仍未达标，必须暂停，输出：

```text
docs/loop_limit_reached.md
```

内容包括：

- 每轮改动
- 每轮分数
- 提升曲线
- 当前瓶颈
- 是否建议继续
- 继续的代价

在用户回复“继续”前，不得再消耗资源做实质性修改。

## 4. Stuck Diagnosis Mode

如果连续 3 轮补强后，全部平均分或变体平均分提升低于 2 分，必须进入 Stuck Diagnosis Mode。

输出：

```text
docs/stuck_analysis.md
```

内容包括：

- 每条失败输入的失败原因
- 是规则覆盖问题、阶段上下文问题、评分问题，还是架构问题
- 是否需要引入新的判断层
- 是否需要重构 signal pipeline
- 是否继续规则补强仍有意义
- 下一轮最多允许尝试 1 个明确补强方向

进入 Stuck Diagnosis Mode 后，不允许继续盲目堆规则。
