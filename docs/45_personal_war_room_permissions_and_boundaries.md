# 个人作战室 V1 权限与安全边界

## 1. V1 权限边界

个人作战室 V1 只读。

不允许：

- 写 SQLite
- 修改记忆
- 自动执行任务
- 接新数据源
- 发送消息
- 控制电脑
- 自动触达外部系统

V1 可以读取本地已有数据，但不能改变系统状态。

## 2. 数据安全

V1 只读取本地已有数据：

- 已存在 JSON 快照
- 已存在审计文档
- 已存在路线图
- 已存在 SQLite 只读结果或导出结果

V1 不读取：

- 浏览器历史
- 邮件
- 通讯录
- 文件系统全盘扫描结果
- 健康数据
- 位置数据
- 外部新闻或情报源

这些能力必须在未来独立阶段中单独设计权限和用户确认机制。

## 3. 用户确认点

进入以下阶段必须用户确认：

- 写入新事件
- 接新数据源
- 外部情报
- 自动执行
- 主动通知产品化
- 多端同步
- 修改 SQLite schema
- 让作战室产生外部副作用

用户确认必须明确到阶段和能力，不能用“继续推进”自动推导授权。

## 4. 防走偏规则

如果个人作战室开始变成以下形态，必须停止并回到 `PROJECT_CONTROL.md` 校验：

- CRUD 后台
- 聊天页
- 任务管理器
- 新闻摘要
- UI 看板
- 数据库管理器
- 外部情报流

防走偏判断：

- 如果页面第一屏是聊天框，失败。
- 如果页面核心是表格，失败。
- 如果页面只展示任务，没有判断，失败。
- 如果页面只展示历史，没有当前局势，失败。
- 如果页面没有反方判断，失败。
- 如果页面没有证据链，失败。

## 5. 阶段边界

阶段 Q 只做规格设计。

即使规格看起来完整，也不能自动进入开发。进入 W1/W2 需要用户单独确认。
