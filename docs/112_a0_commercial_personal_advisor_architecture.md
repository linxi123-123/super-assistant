# A0 商业级个人超级军师总体架构

生成时间：2026-06-11

## 1. 用户最新需求总结

用户明确要求：

- 这不是玩具。
- 不是本地静态页面。
- 不是只做文档和审计。
- 要能处理个人生活、工作、项目管理、金融市场等真实信息。
- 部署在公司服务器。
- 可以联网。
- 可以接入大模型。
- 必须严格保护隐私。
- 敏感信息加密保存在本地/公司服务器。
- 大模型不能直接拿到原始隐私。
- 系统要能提出具体分析和行动建议。

## 2. 旧路线为什么不够

- 不联网无法回答实时市场、新闻、政策、公告、行业变化。
- 不接大模型无法形成高质量综合判断。
- 不保存真实用户上下文，就无法成为固定用户的长期军师。
- 只做作战室页面和快照，无法解决真实高频问题。
- 只做安全边界回复，会安全但无用。
- 过度强调流程和验收，会让项目远离真实价值。

## 3. 新系统核心定义

新系统定义为：

```text
隐私优先的联网大模型个人超级军师系统
```

它的目标不是聊天，而是：

- 长期理解固定用户。
- 记录用户关键事实、目标、项目、承诺、关系、资产、偏好、风险。
- 接入真实外部信息。
- 对用户当前处境做判断。
- 提供具体行动建议。
- 敢于反驳用户。
- 能解释建议依据。
- 能追踪建议结果。
- 能长期修正个人模型。

## 4. 总体六层架构

### 第一层：私密数据保险库 Private Vault

保存：

- 用户身份信息
- 持仓/资产
- 关注标的
- 项目资料
- 工作计划
- 公司内部事项
- 重要关系
- 个人目标
- 长期承诺
- 历史决策
- 行为模式
- 敏感偏好

要求：

- 公司服务器本地加密保存。
- 字段级加密。
- 访问日志。
- 权限控制。
- 敏感字段默认不出本地。
- 所有外发上下文必须先经过隐私治理层。

### 第二层：外部情报层 External Intelligence

允许后续接入：

- 股票行情
- 新闻资讯
- 公司公告
- 财报
- 宏观数据
- 汇率/利率/商品
- 行业信息
- 网页搜索
- 邮件/日历/文档
- 企业内部系统

要求：

- 所有外部数据必须有来源。
- 所有外部数据必须有时间戳。
- 不允许无来源生成事实。
- 不允许假装实时。

### 第三层：隐私治理层 Privacy Governance

负责：

- 敏感字段识别
- PII 检测
- 上下文最小化
- 脱敏
- 授权判断
- 模型调用前审查
- 模型调用日志
- 外发内容审计
- 哪些信息只允许本地使用
- 哪些信息可以脱敏后给模型

核心原则：

- 能不发就不发。
- 能摘要就不发原文。
- 能脱敏就不发明文。
- 高敏信息默认禁止外发。
- 用户授权优先级最高。

### 第四层：大模型推理层 LLM Reasoning

支持：

- OpenAI / Claude / Gemini / DeepSeek / 私有模型等后续接入。
- 模型不直接访问私密数据库。
- 模型只接收经过隐私治理层处理后的任务包。

任务包结构：

```json
{
  "user_query": "",
  "task_type": "",
  "external_context": "",
  "sanitized_user_context": "",
  "constraints": [],
  "output_format": "",
  "risk_rules": []
}
```

### 第五层：本地军师判断层 Local Advisor Judge

负责对大模型输出二次审查：

- 是否泄露隐私
- 是否编造事实
- 是否无来源
- 是否直接给高风险建议
- 是否违背用户长期目标
- 是否忽略风险偏好
- 是否需要反方判断
- 是否需要行动计划
- 是否需要用户确认

输出前必须分清：

- 事实
- 推断
- 建议
- 不确定性
- 风险
- 下一步行动

### 第六层：行动建议与执行层 Action Layer

分级：

- L0：只分析
- L1：给行动建议
- L2：生成计划/待办/草稿
- L3：用户确认后执行
- L4：自动执行

前期只允许 L0-L2。

L3 需要单独设计。

L4 暂不允许。
