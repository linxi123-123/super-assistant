# W6E 阶段报告：第一屏当前语境人工验收准备

生成时间：2026-06-11

## 1. 本阶段目标

W6E 的目标是为用户人工验收 W6D 改造后的第一屏准备验收材料。

本阶段只做：

- 人工验收指南。
- 人工评分模板。
- 人工验收草稿。
- 阶段报告。
- 路线图更新。

W6E 不是开发阶段。

## 2. 已生成哪些人工验收材料

本阶段生成：

- `docs/93_w6e_context_manual_acceptance_guide.md`
- `docs/94_w6e_context_manual_score_template.md`
- `docs/95_w6e_context_manual_acceptance_draft.md`
- `docs/96_stage_w6e_manual_acceptance_prep_report.md`

## 3. 为什么必须人工验收

W6D 的脚本校验已经通过，但 Codex 环境无法完成浏览器手动访问。

更重要的是，W6D 验收不是只看技术是否通过，而是要判断用户是否真的能在第一屏理解：

- 当前阶段。
- 判断生成时间和适用窗口。
- 当前唯一动作。
- 历史模块只是审计。
- 当前不能进入 M0、外部情报或执行代理。

这些属于用户理解质量，必须由用户人工验收。

## 4. 用户应该如何操作

用户应打开：

```text
http://127.0.0.1:8766/app/war_room.html
```

然后按照以下文档评分：

```text
docs/95_w6e_context_manual_acceptance_draft.md
```

评分时不要只确认页面能加载，而要判断第一屏是否真正解决当前语境和阶段时效问题。

## 5. 通过门槛是什么

W6E 人工验收通过标准：

- 总分 `>= 25/30`
- 当前动作 vs 历史动作区分 `>= 6/8`
- 当前阶段清晰度 `>= 4/5`
- 判断时效清晰度 `>= 4/5`

任一硬门槛未达标，都不建议进入下一阶段。

## 6. 如果通过，下一阶段可以是什么

如果 W6E 通过，下一阶段可以作为候选：

- A. M0：长期记忆治理中心规格设计
- B. W7：真实作战室日常输入机制规格设计

但无论选择哪一个，都仍需要用户单独确认。

## 7. 如果不通过，下一阶段应该是什么

如果 W6E 不通过，不允许进入 M0。

建议进入：

```text
W6F：当前语境页面修正
```

W6F 应只针对人工验收发现的问题做修正，不应扩展外部情报、执行代理或长期记忆治理开发。

## 8. 是否触碰禁止事项

本阶段未触碰：

- app 页面。
- JSON 数据。
- SQLite schema。
- SQLite 写入。
- 新数据源。
- 外部网络。
- 外部情报。
- 执行代理。
- M0 记忆治理中心开发。
- npm / 框架 / 第三方库。
- 后端服务。
- 下一阶段开发。

本阶段只新增文档并更新 `tasks/MASTER_ROADMAP.md`。

## 9. 报告结论

在用户完成 `docs/95_w6e_context_manual_acceptance_draft.md` 的人工评分前，不建议进入下一阶段。
