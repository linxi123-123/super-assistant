# W7 日常输入机制验收标准

生成时间：2026-06-11

## 1. 规格通过标准

W7 规格通过必须满足：

1. 明确定义每日作战输入包。
2. 用户每天最少可以只填 1 条核心输入。
3. 字段不超过用户可承受范围。
4. 输入能映射到第一屏五个关键问题。
5. 当前判断和历史审计边界清楚。
6. 不把日常输入自动升级为长期记忆。
7. 不把输入机制设计成聊天记录。
8. 不要求外部数据源。
9. 不要求页面开发。
10. 不要求 SQLite 修改。
11. 不进入外部情报或执行代理。

## 2. 人工评分模板

总分：30 分。

- 输入负担合理性：5
- 字段完整性：5
- 到第一屏映射清晰度：8
- 当前判断 vs 历史审计边界：5
- 长期记忆边界：4
- 阶段禁区清晰度：3

## 3. 通过门槛

W7 通过门槛：

- 总分 `>= 25/30`
- 到第一屏映射清晰度 `>= 6/8`
- 当前判断 vs 历史审计边界 `>= 4/5`
- 长期记忆边界 `>= 3/4`

## 4. 阻断项

出现以下任一情况，W7 不通过：

- 规格要求立刻开发页面。
- 规格要求修改 JSON。
- 规格要求修改 SQLite。
- 规格要求接新数据源。
- 规格让日常输入自动进入长期记忆。
- 规格让用户每天必须写长文本。
- 规格无法生成第一屏五个关键问题。

## 5. 下一阶段候选

如果 W7 通过，下一阶段候选：

- W7D：日常输入机制最小页面/文件原型边界设计。
- M0：长期记忆治理中心规格设计。

无论进入哪个阶段，都需要用户单独确认。
