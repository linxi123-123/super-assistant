# 个人作战室 V1 验收标准

## 1. 规格通过标准

个人作战室 V1 规格通过标准：

1. 页面不是聊天中心。
2. 页面不是 CRUD 后台。
3. 页面第一屏能看到当前局势。
4. 页面能显示当前最重要军师提醒。
5. 页面能显示高价值信号。
6. 页面能显示个人模型假设。
7. 页面能显示承诺与阶段门槛。
8. 页面能显示今日最重要动作。
9. 每个判断能追溯证据链。
10. 每个提醒有反方判断。
11. 每个行动有完成标准。
12. 能跳转或关联 history 审计面板。
13. 不接新数据源。
14. 不写数据库。
15. 不做执行代理。
16. 不引入 npm/框架。
17. 页面打开后用户 30 秒内能知道“现在最该做什么”。

## 2. 不通过标准

以下任一情况出现，即不通过：

- 第一屏是聊天框
- 主要展示数据表
- 只展示历史，不展示当前局势
- 只展示任务，不展示判断
- 无证据链
- 无反方判断
- 无阶段门槛
- 无今日动作
- 用外部情报填补本地数据不足
- 为了页面效果引入 npm 或框架
- 需要用户先解释背景才能理解页面

## 3. 30 秒验收法

用户打开页面后，30 秒内必须能回答：

- 我现在在哪个阶段？
- 当前最重要风险是什么？
- 今日最重要动作是什么？
- 这个动作为什么重要？
- 哪个判断最需要复盘？
- 哪个承诺最需要处理？

如果用户需要打开聊天、翻文档或自行拼接信息才知道答案，则 V1 不通过。

## 4. 军师感验收法

V1 必须体现：

- 判断优先于数据
- 证据优先于结论
- 反方判断优先于迎合
- 阶段门槛优先于扩张
- 今日动作优先于任务堆叠

## 5. 开发前验收

进入开发前必须确认：

- PRD 已通过
- 信息架构已通过
- 数据映射已通过
- 用户工作流已通过
- 权限边界已通过
- 本验收标准已通过
- 用户明确确认进入开发阶段
