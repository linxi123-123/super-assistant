# 市场数据输入规格

生成时间：2026-06-11

## 1. 目标

市场军师必须基于有来源、有时间的数据做判断。

S0 阶段定义两条路线：

- 路线 A：用户手动粘贴行情材料。
- 路线 B：后续受控接入行情 / 新闻数据源。

本阶段只设计，不接外部 API。

## 2. 路线 A：用户手动粘贴行情材料

用户可以粘贴：

- A 股指数涨跌。
- 港股指数涨跌。
- 美股指数涨跌。
- 板块表现。
- 关注股票涨跌。
- 相关新闻摘要。

优点：

- 不接外部网络。
- 立刻可用。
- 风险低。
- 不需要券商账户。

缺点：

- 需要用户手动提供材料。
- 数据质量依赖用户来源。
- 不能自动保证完整性。

## 3. 路线 B：后续受控接入行情 / 新闻数据源

后续可以考虑受控接入：

- 指数行情。
- 个股行情。
- 板块表现。
- 新闻摘要。
- 财报或公告。

注意：

S0 不接入，S0D 若进入也不接外部 API。

## 4. 数据要求

任何市场数据必须包含：

- 来源。
- 时间戳。
- 市场范围。
- 指标或标的名称。
- 原始摘要。

## 5. 禁止事项

- 不允许无来源地生成市场结论。
- 不允许假装实时。
- 不允许把推测当事实。
- 不允许编造新闻。
- 不允许编造用户持仓。
- 不允许直接给买卖指令。

## 6. 无数据时的降级

如果没有实时数据，系统不能只拒绝。

必须说明：

- 当前没有实时行情。
- 需要用户补充哪些材料。
- 将如何分析这些材料。
- 当前不能做什么。
