# S0D 市场军师最小开发计划

生成时间：2026-06-11

注意：本文档不是立即开发授权。是否进入 S0D，需要用户单独确认。

## 1. S0D 目标

让用户在“跟军师说”输入市场类问题时，系统进入市场军师模式。

例如：

```text
今天股市怎么样
```

系统应输出结构化市场边界与个性化关注提醒，而不是项目流程话。

## 2. S0D 只允许做

如果用户确认进入 S0D，只允许：

1. 在 `app/app.js` 中识别市场类问题。
2. 读取 `data/user_market_profile.local.json`。
3. 如果没有实时数据，给出结构化边界回复。
4. 如果用户在输入中粘贴了行情材料，基于用户提供内容做简短分析。
5. 显示关注列表和持仓相关提醒。
6. 不接外部 API。
7. 不接券商账户。
8. 不给买卖指令。
9. 不写 SQLite。
10. 不引 npm/框架。

## 3. S0D 禁止事项

禁止：

- 自动交易。
- 预测涨跌。
- 伪造实时行情。
- 编造用户持仓。
- 编造新闻。
- 直接推荐买入 / 卖出。
- 接券商账户。
- 接外部 API。
- 写 SQLite。
- 引入 npm / 框架。

## 4. 最小验收标准

S0D 最小验收：

- 用户问“今天股市怎么样”时，系统进入市场军师模式。
- 系统显示 A 股 / 港股 / 美股三个范围。
- 系统显示用户关注标的。
- 如果 holdings 为空，明确说明“未确认持仓”。
- 系统不编造实时行情。
- 系统要求用户补充行情材料或使用后续受控数据源。
- 系统不直接给买卖指令。

## 5. 结论

S0D 应优先验证市场军师最小闭环是否让用户感觉真实有用，而不是继续堆作战室页面。
