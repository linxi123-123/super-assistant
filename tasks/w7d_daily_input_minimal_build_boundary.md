# W7D 日常输入机制最小实现边界草案

生成时间：2026-06-11

注意：本文档不是开发任务，只是 W7 通过后可能进入 W7D 的边界草案。

## 1. W7D 进入条件

只有在 W7 规格通过，且用户单独确认进入 W7D 后，才允许进入最小实现。

## 2. W7D 可能允许

W7D 若启动，最多允许：

- 最小页面或本地文件原型。
- 只读展示每日作战输入包字段。
- 不写 SQLite。
- 不接新数据源。
- 不引入 npm / 框架。
- 不做聊天。
- 不做同步。
- 不进入外部情报或执行代理。

## 3. W7D 明确禁止

W7D 禁止：

- 自动写入长期记忆。
- 修改 SQLite schema。
- 接外部数据源。
- 使用外部网络。
- 做后端服务。
- 做复杂 UI。
- 做执行代理。
- 做登录 / 同步 / 上传。

## 4. W7D 验收方向

W7D 只验证：

- 用户是否能理解每天填什么。
- 输入是否能生成作战室当前判断。
- 当前判断和历史审计是否仍然分开。
- 输入负担是否足够低。

W7D 不验证长期记忆治理，不验证外部情报，不验证执行代理。
