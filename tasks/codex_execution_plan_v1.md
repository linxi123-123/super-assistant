## 阶段 1：本地原型

目标：

做一个本地可运行的“个人局势军师 V1”。

建议技术：

- 前端：Next.js / React
- 后端：Node.js 或 Python
- 数据库：SQLite 起步，后续迁移 PostgreSQL
- 向量：V1 可先不做，先结构化存储
- 模型：通过 API 调用

核心页面：

1. 作战室 Dashboard
2. 事件输入
3. 目标与项目
4. 记忆中心
5. 个人模型
6. 信号中心
7. 军师提醒
8. 决策推演

## 阶段 1 开发任务拆分

### Task 1：初始化应用

完成标准：

- 应用可本地启动。
- 有基础路由和布局。
- 首页不是聊天框，而是作战室 Dashboard。

### Task 2：实现数据模型

完成标准：

- 建立 UserProfile、Goal、Project、Event、Memory、PersonalModelHypothesis、Signal、Touchpoint、Action。
- 支持创建、读取、更新、删除。

### Task 3：事件输入与事件列表

完成标准：

- 用户可以输入每日事件。
- 系统要求选择或推断事件类型。
- 事件记录包含来源、重要性、置信度、敏感度。

### Task 4：每日局势快照

完成标准：

- 根据当天事件生成局势快照。
- 输出事实、推测、建议三部分。
- 生成明日最重要动作。

### Task 5：候选记忆提取

完成标准：

- 从事件中生成候选记忆。
- 显示证据和置信度。
- 用户可确认、修改、删除。

### Task 6：个人模型假设

完成标准：

- 根据事件和记忆生成模型假设。
- 每条假设包含证据、反例、置信度和验证计划。

### Task 7：信号识别

完成标准：

- 识别偏离、机会、风险、承诺、状态、模式信号。
- 每个信号有评分。
- 可判断是否触达。

### Task 8：主动军师提醒

完成标准：

- 根据高价值信号生成提醒。
- 提醒包含原因、观察、目标关联、风险/机会、建议动作、不行动后果。
- 用户可反馈准确性。

### Task 9：决策推演模式

完成标准：

- 用户输入重大问题。
- 系统按 17 步决策链路输出。
- 结果保存到决策日志。

### Task 10：权限与记忆审计

完成标准：

- 记忆可查看来源和证据。
- 记忆可删除。
- 记忆可设置是否允许用于建议。
- P2 以上敏感信息默认需要确认。

## 阶段 1 不做

- 全天候监听。
- 自动读取全部聊天软件。
- 高风险自动执行。
- 手机端。
- 复杂 Agent 编排。
