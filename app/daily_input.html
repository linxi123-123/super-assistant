Daily War Room Input / W7D

每日作战输入包

只读原型：帮助用户理解每天最少输入什么，以及这些输入如何生成作战室当前判断。

当前阶段 W7D：日常输入机制最小实现

边界

本页不保存、不编辑、不写入 JSON 或 SQLite，不接新数据源，不进入外部情报、执行代理或长期记忆治理中心开发。

Minimum Daily Input

每天最少只需要 1 条核心输入

可填其一

今日真实事件 或 当前最想判断的问题

必须带边界

当前限制 / 禁止事项必须清楚，否则不得生成强判断。

input_date

输入日期

记录输入属于哪一天。

示例：2026-06-11

key_event

今日真实事件

当天最重要的真实事件或事实变化。

示例：W6E 已通过，用户选择进入 W7D。

current_question

当前判断问题

今天最想让作战室帮助判断的问题。

示例：下一步应该做最小输入原型，还是先补记忆治理？

project_state_change

项目状态变化

项目、目标、关系或资源状态发生了什么变化。

示例：第一屏已经能区分当前判断与历史审计。

committed_action

今日承诺动作

用户已经承诺或准备今天完成的动作。

示例：今天只验证每日输入包是否足够轻。

constraints_or_forbidden

限制与禁止事项

当前明确不能做什么。

示例：不写数据库，不接外部情报，不进入执行代理。

evidence_note

证据说明

输入来源或用户说明，用于审计。

示例：来自用户在 W7 后的明确选择。

sensitivity

敏感度

标记输入敏感度：low / medium / high。

规则：high 不得自动进入长期记忆或外部传输。

use_for_today_judgment

是否用于今日判断

yes / no / unclear。

规则：unclear 不得生成强判断，需用户澄清。

Mapping

如何映射到作战室第一屏

当前局势一句话

来自 key_event、current_question、project_state_change。

当前最大矛盾

来自 current_question、project_state_change、constraints_or_forbidden。

军师直接判断

只基于允许参与今日判断的输入，并保留反方判断。

今日唯一动作

来自 committed_action 和当前问题；多个动作冲突时先澄清。

不要做清单

优先显示 constraints_or_forbidden 和阶段硬边界。

当前判断与历史审计边界

进入当前判断

仅限 use_for_today_judgment = yes，且输入足以支撑今日判断的内容。

只进入历史审计

未确认、敏感、证据不足或标记为 no 的输入只用于审计和复盘。

不得自动长期化

在 M0 完成前，日常输入不得自动升级为长期事实、偏好或个人模型。
