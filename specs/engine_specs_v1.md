# 核心引擎规格 v1

## 1. 事件引擎

职责：

- 接收用户输入或外部数据。
- 判断输入代表事实、判断、承诺、担忧、状态还是机会。
- 生成结构化事件。

输入：

- 用户每日复盘。
- 军师对话。
- 手动录入的外部情报。

输出：

- Event。

V1 验收：

- 任意输入必须先进入事件层。
- 事件必须包含来源、类型、重要性、置信度、敏感度。

## 2. 记忆引擎

职责：

- 从事件和每日摘要中提取候选记忆。
- 判断是否应该进入长期记忆。
- 管理证据链、置信度、过期和用户确认。

输出：

- Memory。

V1 验收：

- 每条记忆能追溯到事件。
- 敏感记忆必须用户确认。
- 用户可以删除或禁止用于建议。

## 3. 个人模型引擎

职责：

- 基于事件和记忆生成阶段性假设。
- 识别行为模式、目标偏离、风险偏好和当前瓶颈。

输出：

- PersonalModelHypothesis。

V1 验收：

- 模型表达必须是“假设”，不是绝对结论。
- 每条假设必须有证据、反例、置信度和验证计划。

## 4. 信号识别引擎

职责：

- 从目标、项目、事件、记忆、个人模型中发现高价值信号。
- 对信号进行评分。

输出：

- Signal。

V1 验收：

- 支持偏离、机会、风险、承诺、决策、状态、模式七类信号。
- 每个信号必须有可行动性评分和打扰成本评分。

## 5. 主动触达引擎

职责：

- 判断哪些信号值得提醒用户。
- 生成有证据、有行动建议的军师提醒。

输出：

- Touchpoint。

V1 验收：

- 不重复提醒同一低价值信号。
- 每次触达必须说明原因、证据、目标关联、建议动作和不行动后果。

## 6. 决策推演引擎

职责：

- 对重大问题进行结构化推演。

输出结构：

```text
问题定义
真实目标
当前筹码
关键约束
可选路径
收益
代价
成功概率
失败代价
机会成本
反方观点
推荐方案
24 小时动作
7 天动作
30 天动作
验证指标
复盘时间
```

V1 验收：

- 不能只输出结论。
- 必须输出判断链路、行动计划和验证标准。

## 7. 情报引擎

V1 先支持手动或半自动录入外部情报。

职责：

- 判断外部情报是否和用户目标相关。
- 提取机会、风险、建议动作。

V1 验收：

- 情报不能只是新闻摘要。
- 必须回答“和用户有什么关系”。
